🌟 Spotlight
Spotlight is a real‑world connection platform that helps people meet nearby based on shared intent — like coffee, study, or casual conversations.
It focuses on offline interaction, trust, and privacy, instead of endless chatting or swiping.

🚀 Overview
Most platforms keep users online.
Spotlight is built to take users offline.

Users can:

Check in at a location

Discover nearby users with similar intent

Send connection requests

Meet after mutual acceptance

Build trust through real-world interactions

🎯 Core Idea
Less chatting. More real meetings.

Spotlight removes:

Swiping culture

Fake profiles

Endless messaging

And focuses on:

Intent-based connections

Real-life meetups

Accountability and trust

🧩 Features
👤 User Accounts & Profiles
Secure authentication (email, password, Google login)

Profile setup (username, bio, gender, DOB)

Interest-based “vibe” tags

Profile management

📍 Spotlight / Check-In System
Users check in to their current location

Add intent (coffee, study, walk, etc.)

Temporary visibility (auto-expiry)

Location-based discovery

🔍 Nearby Discovery
View users within a defined radius

Real-time availability

Intent-based matching

🤝 Matching System
Send and receive connection requests

Accept or decline requests

One active match at a time

Match ends manually with reason

🧭 Real-World Coordination
“I’m reaching” indicator

“I reached” confirmation

No chat system (intentional design choice)

⭐ Trust & Feedback System
Post-meeting rating (1–10)

Optional feedback comments

Trust score calculation

Visible recent feedback

🚨 Safety & Reporting
Report users for misconduct

Block users

Admin moderation system

Feedback-based trust system

🛠️ Admin Panel
View users and platform stats

Manage reports

Block/unblock users

Delete accounts

Export reports (CSV)

🗄️ Database Structure
Main tables:

users → authentication & account data

profiles → user details

spotlights → active check-ins

requests → connection requests

matches → active/ended matches

reviews → feedback & ratings

reports → user/app reports

🔐 Security Approach
Password hashing

Role-based access control (admin/user)

Server-side validation

Request validation

Anti-spam & cooldown logic
